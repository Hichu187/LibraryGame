# Quy tắc làm việc

## Ngôn ngữ
Trao đổi, review, comment trong code — tiếng Việt.

## Code review
- Liệt kê vấn đề kèm mức độ ưu tiên: **Cao / Trung bình / Thấp**
- Hỏi muốn fix cái nào trước thay vì tự fix tất cả
- Khi sửa: chỉ sửa đúng phần được yêu cầu, không thêm tính năng ngoài scope

## Triết lý
Project ưu tiên prototype nhanh. Ba dòng code giống nhau tốt hơn một abstraction non-obvious. Không thêm error handling, fallback, hay validation cho tình huống không thể xảy ra.
